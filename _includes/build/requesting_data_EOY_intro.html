Temporary Change to Production Requests

During January 2021, production users have to change how they request data. Requests must be sent to the /Group endpoint and must include a new identifier: "runouts". All the claims returned will have a service date of 12/31/2020 or earlier. Any requests to the /Group (without the "runouts" identifier) or /Patient endpoints will fail and result in an error message.

These changes are in place to ensure that BCDA does not return any data not assigned to your organization during the transition between performance years. Please see the full details in the End of Year Requests section below.

To obtain any data for your beneficiaries during January 2021, you must make a request to the /Group endpoint with the "runout" identifier. You may want to do this to retrieve data during the end of year reassignment period or simply to retrieve your runouts files. The claims returned when using "runout" will be filtered by their serviceData. You will not receive any claim with a service date of 12/31/20 or before.

When will these changes end?

We expect these changes to be in effect until BCDA retrieves the January 2021 attribution file from CMS. This should be no later than January 31, 2021. After this time, requests to /Patient and /Group using `all` will return to their normal state of operation. In addition, you can continue to use /Group with `runout` to retrieve your February runouts data. BCDA will update the Google Group when you can begin using `all` again.

What changes does my ACO need to make?

In January 2021:

  1. To request any claims, users will need to use the `runout` identifier with the /Group endpoint.
  2. Requests to the /Group and /Patient endpoints without the `runout` identifier will result in an error message.
  3. When using the `runout` identifier, data returned by the API will be restricted to claims with a service date of 12/31/2020 or earlier.

Between February and April 2021:

  1. Requests to the /Group and /Patient endpoints can again use the `all` identifier, as normal and will return claims with a service date in 2021.
  2. To request runouts data, users can also make a call using the `runout` identifier with the /Group endpoint.
  3. When using the `runout` identifier, data returned by the API will be restricted to claims with a service date of 12/31/2020 or earlier.

List of Requests

During January, you can make a few different unfiltered requests for data, all of them must be to the /Group endpoint and include the 'runout identifier:

  1. Request all three Resource Types at once
  2. Request any two Resource Types
  3. Request a single Resource Type

Steps

Each request will follow the same four step process:

  1. Obtain an access token
  2. Start a job to acquire data
  3. Check the job status
  4. Download the data
